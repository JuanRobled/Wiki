Pruebas

Pruebas de usabilidad

testGetTeamById()

Se espera que se obtenga un equipo por su ID y que el resultado no sea nulo, y sea igual al equipo esperado.

testGetTeamByIdThrowsExceptionWhenTeamNotExists()

Se espera que se lance una excepción TeamNotExistsException cuando se intenta obtener un equipo que no existe en la base de datos.

testFindAll()

Se espera que se obtengan todos los equipos en la base de datos y que el resultado no sea nulo, y que el tamaño de la lista de equipos sea igual a 1, y que el equipo obtenido sea igual al equipo esperado.

testAddTeam()

Se espera que se agregue un equipo a la base de datos, que se guarde correctamente en el repositorio, y que se llame al método addTeamToUser() del servicio de usuarios con los parámetros correctos.

testDeleteTeam()

Se espera que se borre un equipo de la base de datos, que se llame al método delete() del repositorio con el equipo correcto, y que el equipo sea eliminado de la lista de equipos del usuario.

testGetUsersByTeamId()

Se espera que se obtengan todos los usuarios que pertenecen a un equipo por su ID, y que el resultado no sea nulo, y que el tamaño de la lista de usuarios sea igual a 1, y que el usuario obtenido sea igual al usuario esperado.

testUpdateTeam()

Se espera que se actualice el nombre de un equipo en la base de datos, y que el nombre del equipo actualizado sea igual al nombre esperado.

testGetTeamsWhereUserIsNotAt()

Se espera que se obtengan todos los equipos a los que un usuario no pertenece, y que el resultado no sea nulo, y que la lista de equipos obtenidos no contenga al usuario esperado.

Pruebas unitarias

shouldThrowUserNotExistsException()

Esta prueba verifica si se lanza una excepción UserNotExistsException cuando se intenta obtener un usuario por un ID que no existe en el repositorio. El resultado esperado es que se lance la excepción UserNotExistsException.

shouldAddUser()

Esta prueba verifica si se puede agregar un usuario correctamente. El resultado esperado es que el usuario agregado sea igual al usuario esperado, sin lanzar ninguna excepción.

shouldThrowUsernameAlreadyExistsException()

Esta prueba verifica si se lanza una excepción UsernameAlreadyExistsException cuando se intenta agregar un usuario con un nombre de usuario que ya existe en el repositorio. El resultado esperado es que se lance la excepción UsernameAlreadyExistsException.

shouldThrowEmailAlreadyExistsException()

Esta prueba verifica si se lanza una excepción EmailAlreadyExistsException cuando se intenta agregar un usuario con una dirección de correo electrónico que ya existe en el repositorio. El resultado esperado es que se lance la excepción EmailAlreadyExistsException.

shouldDeleteUser()

Esta prueba verifica si se puede eliminar un usuario correctamente. El resultado esperado es que el usuario sea eliminado del repositorio y se lance una excepción UserNotExistsException al intentar obtenerlo nuevamente por su ID.
