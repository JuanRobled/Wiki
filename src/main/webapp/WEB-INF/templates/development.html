Desarrollo

El desarrollo del proyecto se llevó a cabo utilizando un sistema de control de versiones llamado Git, y se utilizó GitHub como plataforma de alojamiento remoto para el repositorio de Git.

Sistema de control de versiones

Git

Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear y controlar los cambios en el código fuente de un proyecto. GitHub, por su parte, es una plataforma en línea que proporciona servicios de alojamiento para proyectos de software utilizando Git como sistema de control de versiones. En GitHub, los desarrolladores pueden trabajar juntos en proyectos y compartir su trabajo con otros colaboradores, lo que facilita la colaboración en equipo y el seguimiento de los cambios realizados en el proyecto.

Github

La utilización de GitHub para manejar las versiones del código fuente y llevar un registro de los cambios realizados en el mismo es una práctica común en el desarrollo de software. Al utilizar esta plataforma, los desarrolladores pueden trabajar de forma colaborativa en un mismo proyecto, sin tener que preocuparse por perder información o versiones anteriores del código.

La creación de ramas (branches) es una funcionalidad clave de GitHub, ya que permite a los desarrolladores trabajar en cambios específicos sin afectar la rama principal (main branch) del proyecto. Cada rama es una copia del código en su estado actual, lo que permite a los desarrolladores trabajar en paralelo y experimentar con nuevas ideas sin comprometer el código existente.

Una vez que un desarrollador ha terminado de trabajar en sus cambios, pueden enviar una solicitud de extracción (pull request) al equipo principal para revisar y fusionar sus cambios con la rama principal (main branch). Esto permite una revisión y aprobación cuidadosa del trabajo realizado por cada desarrollador, lo que asegura que el código que se fusiona en la rama principal es de alta calidad y no tiene errores o conflictos con otras partes del código.

En resumen, GitHub es una herramienta valiosa para el control de versiones y la gestión de proyectos de software, ya que permite una colaboración eficiente y organizada entre los miembros del equipo. La creación de ramas y las solicitudes de extracción son características importantes que ayudan a garantizar que el código se desarrolle de manera segura y confiable.
